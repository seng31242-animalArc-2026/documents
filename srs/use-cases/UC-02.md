# UC-02 — Book Appointment

Use Case ID: UC-02

Use Case Name:
Book Appointment

Actors:
Animal Owner

Preconditions:
- Animal must already be registered.

Postconditions:
- Appointment is booked successfully.

Main Flow:
1. User opens appointment section.
2. Selects animal profile.
3. Chooses available date and time.
4. Confirms booking.
5. System stores appointment details.

Alternative Flow:
- If selected slot is unavailable, system suggests another time.

Business Rules:
- One appointment can only belong to one animal.
