# Animal Arc - Team Working Agreement

## Project Details

**Project Name:** Animal Arc  
**Course:** SENG 31242 - System Design Project  
**University:** University of Kelaniya  
**Academic Year:** 2026  

## Communication Channel

The primary communication channel for the team will be WhatsApp. Important decisions, meeting outcomes, task updates, and submission-related information must also be documented in GitHub.

## Working Hours

The team agrees to work on the project during the following common time slots:

- Weekdays: 7:00 PM - 10:00 PM
- Weekends: 9:00 AM - 12:00 PM or based on team availability

If a member is unavailable during the agreed working hours, they should inform the team in advance.

## GitHub Usage Rules

- All project tasks must be tracked using GitHub Issues.
- Each issue must have a clear title, description, assignee, label, milestone, and estimate.
- Work must be completed in a separate branch.
- The `main` branch must not be updated directly.
- Pull Requests must be reviewed by at least one team member before merging.
- The GitHub Project board must be updated regularly.

## Branch Naming Rules

Use the following branch naming format:

- `draft/srs-chapter1`
- `draft/use-cases`
- `draft/sds-architecture`
- `draft/meeting-notes`
- `fix/issue-number`

## Commit Message Rules

The team will follow Conventional Commits.

Examples:

- `docs(srs): add project scope section`
- `feat(requirements): add emergency alert use case`
- `fix(diagram): correct actor relationship in use case diagram`
- `style(report): format SRS headings`
- `chore(repo): update gitignore file`

## Meeting Rules

- Sprint stand-ups will be held every 2-3 days.
- Each member must provide updates using the following format:
  1. What did I complete?
  2. What am I doing next?
  3. Are there any blockers?
- Meeting notes must be documented within 24 hours.
- Important meeting decisions must be added to GitHub Wiki or the documents repository.

## Task Responsibility

Each member is responsible for completing their assigned GitHub issues before the due date. If a task is delayed, the member must inform the Team Lead as early as possible.

## Review Process

Before finalising any document or diagram:

- At least one team member must review it.
- Spelling and grammar must be checked.
- The document must match the project scope.
- The output must follow the course guideline.

## Conflict Handling

If there is a disagreement or delay, the issue should first be discussed within the team. If it cannot be resolved, the Team Lead will contact the academic supervisor for guidance.

## Agreement

All team members agree to follow this working agreement throughout the SENG 31242 System Design Project.
