## Role Summary

### Team Lead
The Team Lead is responsible for coordinating the team, managing sprint progress, tracking GitHub issues, following up deadlines, and communicating important updates to supervisors.

### Documentation Lead
The Documentation Lead is responsible for maintaining the SRS, SDS, meeting notes, report formatting, proofreading, and ensuring all documents are consistent.

### Requirements & Diagram Lead
The Requirements & Diagram Lead is responsible for use case diagrams, use case descriptions, activity diagrams, sequence diagrams, and system analysis documentation.

### Research & Prototype Lead
The Research & Prototype Lead is responsible for fact-finding preparation, stakeholder interview/survey materials, competitive analysis, UI wireframes, and prototype-related artefacts.

## Team Agreement

All team members agree to:
- Complete assigned GitHub issues before the due date.
- Update task progress on the GitHub Project board.
- Attend project meetings and stand-ups.
- Communicate blockers early.
- Review each other's work before merging into the main branch.
- Keep all project artefacts updated in the relevant GitHub repositories.
